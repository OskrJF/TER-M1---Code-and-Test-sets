Here are three effective methods for finding lattice concepts in a formal context with fewer than 20 objects and attributes:

## 1. Direct Enumeration with Closure Operators

For small contexts, you can systematically enumerate all possible object sets and compute their concept closures. Start with the empty set and all singleton object sets, then apply the derivation operators:
- For object set A, find A' (all attributes shared by objects in A)
- Then find A'' (all objects that have all attributes in A')
- If A = A'', then (A, A') forms a concept

Continue this process for all possible combinations of objects, pruning redundant computations by recognizing when larger sets will produce the same closures as smaller ones.

## 2. Next-Closure Algorithm (Ganter's Algorithm)

This is the most computationally efficient approach for contexts of this size. The algorithm generates concepts in lexicographic order by:
- Starting with the bottom concept (all objects, their common attributes)
- Iteratively finding the "next" concept by systematically modifying the intent (attribute set)
- Using a clever closure operation that ensures each concept is generated exactly once

The algorithm is particularly elegant because it avoids generating duplicate concepts and has predictable memory requirements.

## 3. Attribute-Incremental Construction

Build the concept lattice incrementally by processing one attribute at a time:
- Start with the lattice containing only the top concept (empty object set, all attributes)
- For each new attribute, identify which existing concepts need to be split
- Create new concepts by intersecting object sets of existing concepts with the object set of the new attribute
- Update the lattice structure by adding appropriate edges

This method is intuitive and allows you to visualize how each attribute contributes to the final lattice structure. It's particularly useful when you want to understand the role of individual attributes in forming the concept hierarchy.

For contexts with fewer than 20 objects and attributes, any of these methods will complete quickly, with the next-closure algorithm typically being the fastest for implementation in software.
