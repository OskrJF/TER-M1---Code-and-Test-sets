## What is a Formal Concept?

A **formal concept** is a pair (A, B) where:
- **A ⊆ G** is called the **extent** (set of objects)
- **B ⊆ M** is called the **intent** (set of attributes)
- They satisfy: A' = B and B' = A

The **derivation operators** work as follows:
- A' = {attributes shared by ALL objects in A}
- B' = {objects that have ALL attributes in B}

3. **Bottom Concept**: (∅, All attributes) - no objects have all attributes

4. **Concept Structure**: Each concept represents a maximal group of objects that share exactly the same set of attributes

## How the Algorithm Works

The extraction process:
1. **Generate all possible attribute combinations** (power set of attributes)
2. **For each attribute set B**, find all objects that have ALL these attributes (B')
3. **For the resulting object set A**, find all attributes shared by ALL these objects (A')
4. **Check closure**: If A' = B, then (A, B) is a formal concept
5. **Remove duplicates** and organize the results

The concepts form a **lattice structure** where concepts can be ordered by inclusion of their extents (or dually, by inclusion of their intents).
