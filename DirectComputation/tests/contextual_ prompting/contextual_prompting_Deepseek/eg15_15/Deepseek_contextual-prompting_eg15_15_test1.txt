### Understanding the Problem

First, let's clarify what we're dealing with:

- **Formal Context**: This is a table that shows the relationship between objects (rows) and attributes (columns). In our case, we have objects O1 to O15 and attributes Attr1 to Attr15. A "Yes" means the object has that attribute, and "No" means it doesn't.
  
- **Formal Concept**: In Formal Concept Analysis (FCA), a formal concept is a pair (A, B), where:
  - A is a set of objects (called the extent).
  - B is a set of attributes (called the intent).
  - Every object in A has all attributes in B, and every attribute in B is possessed by all objects in A.
  - There's no larger set of objects than A that all have attributes B, and no larger set of attributes than B that all objects in A have.

- **Concept Lattice**: This is a hierarchy of all formal concepts from the context, ordered by a subconcept-superconcept relation.

Given that our context is small (15 objects and 15 attributes), we can manually extract all formal concepts without needing complex algorithms.

### Step 2: Find All Formal Concepts

To find all formal concepts, we can:

1. Start with an attribute or an object and find all objects that share that attribute (or all attributes shared by that object).
2. Then, for that set of objects, find all common attributes.
3. Check if this pair (set of objects, set of attributes) is maximal in both directions.

This process can be systematic:

- **From Attributes**: For each subset of attributes, find all objects that have all those attributes (intent to extent), then find all attributes common to those objects (extent back to intent). If this brings us back to the original attributes, it's a concept.
  
- **From Objects**: Similarly, start with a set of objects, find their common attributes, then find all objects with those attributes.

Given the size, starting with single attributes might be manageable.
