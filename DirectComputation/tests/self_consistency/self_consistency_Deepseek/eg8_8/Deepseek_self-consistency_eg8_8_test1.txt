### Formal Concept Analysis (FCA) Basics

In FCA, a **formal concept** is a pair (A, B), where:
- A is a set of objects (called the extent).
- B is a set of attributes (called the intent).
- Every object in A has all attributes in B, and every attribute in B is possessed by all objects in A.
- There is no larger set of objects than A that have all attributes in B, and no larger set of attributes than B that are shared by all objects in A.

The set of all formal concepts forms a complete lattice called the **concept lattice**.

### Steps to Extract All Formal Concepts

To find all formal concepts, we can follow these steps:

1. **List all possible subsets of objects**: For each subset, find the common attributes shared by all objects in the subset (this is the intent).
2. **List all possible subsets of attributes**: For each subset, find all objects that have all those attributes (this is the extent).
3. **Find pairs (A, B)** where the extent of B is A, and the intent of A is B. These are the formal concepts.

However, enumerating all subsets is computationally intensive. Instead, we can use a more systematic approach like the **NextClosure algorithm** or **Ganter's algorithm** to generate all concepts efficiently. For manual computation, we'll proceed step by step.

### Manual Computation Approach

Given the small size of the context, we can manually compute the concepts by:

1. Starting with the set of all objects and finding their common attributes.
2. Then, removing one object at a time and finding the new intents.
3. Similarly, starting with the set of all attributes and finding their extents.
4. Combining these to find all valid (extent, intent) pairs.

However, a more practical manual method is to:

- For each object, find its intent (attributes it has), then find all other objects that share this intent.
- For combinations of objects, find their common attributes and corresponding extents.

This will generate all concepts.
