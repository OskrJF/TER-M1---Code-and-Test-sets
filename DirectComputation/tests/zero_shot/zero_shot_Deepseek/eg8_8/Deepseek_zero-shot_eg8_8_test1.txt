### Understanding the Problem

First, let's clarify what we're dealing with:

- **Formal Context**: This is a table that shows the relationship between objects (rows) and attributes (columns). In our case, the objects are O1 to O8, and the attributes are a1 to a8. A "Yes" indicates that the object has the attribute, and "No" means it doesn't.
  
- **Formal Concept**: In Formal Concept Analysis (FCA), a formal concept is a pair (A, B), where:
  - A is a set of objects (called the extent),
  - B is a set of attributes (called the intent),
  - Every object in A has all attributes in B,
  - Every attribute in B is possessed by all objects in A,
  - There's no larger set of objects than A that all have attributes B, and
  - There's no larger set of attributes than B that all objects in A have.

- **Concept Lattice**: This is a hierarchical structure that shows all formal concepts and their relationships (like which concepts are more general or more specific than others).

**Note**: In practice, algorithms like NextClosure or Ganter's algorithm are used to compute all concepts efficiently, but here we'll do it manually.
